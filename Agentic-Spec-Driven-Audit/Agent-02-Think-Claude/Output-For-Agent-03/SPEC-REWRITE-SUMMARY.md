# Claude → Gemini: Spec Rewrite Summary

## What Was Done

The original Causal Engine v1.0 spec (517 lines, generated by Perplexity Computer) was rewritten into a tighter, implementation-ready specification (568 lines). The rewrite is at `CONTEXT-BASED/Spec/causal-engine-v1.0-spec.md`.

## Why It Was Done

The original spec had 7 systematic problems, all from one root cause: it was written from Pearl's textbook perspective instead of from a v1-engineering perspective.

## The 7 Problems Fixed

| # | Problem | What Changed |
|---|---------|--------------|
| 1 | Used `P(Y\|do(X=x))` notation (distributional) | Replaced with `Y_{do(X=x)}` (deterministic point value) throughout |
| 2 | Included adjustment logic (backdoor/frontdoor) | Moved to v1.1 — mutilation IS the v1 answer |
| 3 | Identifiability gate was vacuous (always true under v1 assumptions) | Reframed as model completeness validation |
| 4 | 7 query types when only 3 are real | Reduced to: observational, interventional, conditional interventional |
| 5 | B3 (Frontdoor) required hidden confounders; B7 (Napkin) was semi-Markovian | Replaced with v1-scoped benchmarks (Simple Chain, Diamond Graph) |
| 6 | Three notation styles coexisted | Single notation throughout |
| 7 | "Mathematically legitimate causal inference engine" overclaimed | Anchor: "deterministic SCM intervention executor" |

## 5 Additional Fixes

| # | Fix |
|---|-----|
| 8 | Removed `'adjustment'` from CausalQuery type union |
| 9 | Fixed B1 explanation — engine passes via mutilation, not adjustment |
| 10 | Linear-only for v1.0, nonlinear deferred to v1.1 (no "primary/secondary" ambiguity) |
| 11 | Added explicit deprecation note for `propagateIntervention()` / 0.7 decay |
| 12 | Noted `structuralEquationsJson: Array<Record<string, unknown>>` as first typing task |

## Key Interfaces Defined

```typescript
interface StructuralEquation {
  variable: string;
  parents: string[];
  coefficients: Record<string, number>;
  intercept: number;
}

interface TypedSCM {
  variables: SCMVariable[];
  equations: StructuralEquation[];

interface CausalQuery {
  type: 'observational' | 'interventional';
  treatment: string;
  outcome: string;
  doValue: number;
  conditions?: Record<string, number>;
}
```
