Section 01

Competitive Landscape

The screenshot you shared — T3 Code — is a useful reference because it exposes the dominant AI tool UI pattern of 2025: two-pane with session sidebar. Understanding what this pattern does well and where it fails is the starting point for differentiation.

Key Signal: The move away from pure chat

Chat-as-primary-interface is becoming a commodity. The trend in 2025–26 is toward orchestration UI — users direct work, not just converse. Cursor's parallel agents, Notion's inline AI, and Claude's Projects all reflect this. A flat session list is a 2023 pattern.

Key Signal: Session continuity matters for research tools

Research shows conversation categorization helps users find specific sessions 30% faster. For a scientific tool where sessions represent ongoing investigations — not throwaway chats — semantic grouping (by experiment, domain, or law candidate) is a critical UX gap in current tools.

Key Signal: Product-as-hero on landing pages

Top SaaS pages in 2025 embed the actual product — not illustrations of it — in the hero. Notion, Linear, and Framer all show real UI before users scroll. MASA's hero should be the synthesis engine in motion, not a description of it.

Key Signal: Dark + minimal wins for dev/research tools

Dark mode with generous whitespace and a single restrained accent color is the dominant aesthetic for serious research and developer tools. This aligns exactly with the Wabi-Sabi design system — the system is right; the implementation needs to catch up.

Section 02

Sidebar Redesign

The T3 Code pattern — project folder → flat session list with timestamps — is the baseline to beat. The core problem with this pattern for MASA is that all sessions are treated as equal. A "Hey" session from 22 minutes ago has the same visual weight as a long-running synthesis run on causal drug discovery. That's wrong for a scientific tool.

Current Pattern (T3 Code style) — Problems

┌──────────────────────────────┐ │ ▾ Synthetic-Mind │ │ Hey 22m ago │ ← No context, no type │ Chat session 2 1h ago │ ← All sessions identical │ Chat session 3 3h ago │ ← No status signals │ │ │ ▾ standalone-tools │ │ ... │ └──────────────────────────────┘ Problems: flat, context-free, no session type, no scientific status, no grouping by domain.

MASA Sidebar — Proposed Pattern

The sidebar should reflect that MASA sessions are scientific investigations, not chat threads. Three structural changes: semantic grouping by mode, causal status badges, and pinned active experiments at the top.

Sidebar Design Principles

1. Mode-grouped, not date-sorted

Sessions live under their interface mode (Hybrid, Causal Chat, Legal). A scientist doesn't think "what did I do 22 minutes ago?" — they think "which synthesis runs are active?" Mode grouping reflects how scientists actually organize work.

2. Status as first-class signal

Running ●, novel ◆, conflict ⚠, complete ✓ — the sidebar should tell you the epistemic state of every session at a glance. This is impossible in a timestamp-only list. Inspired by Linear's status-first design.

3. Pinned active experiments at top

Long-running synthesis runs should be pinnable above the mode groups. This echoes the Claude.ai "Projects" concept but scoped to experiment runs rather than conversation threads.

4. Wabi-Sabi width and restraint

220–240px max. Stone texture on the sidebar background. Bamboo-colored active state. Session names should be truncated with an ellipsis — not wrapped. The sidebar is navigation, not reading material.

"70% of users rely heavily on sidebars for quick navigation. Categorization helps users find specific sessions 30% faster." — UX Practices for Sidebar Design, 2025

Card Design Principles

Mode identity in the header

Every card opens with its mode (hybrid / chat / legal), novelty/status badge, Pearl ladder level, and session age. Not the session title alone. The header communicates what kind of scientific work this is.

Stage progress for Hybrid

Observe → Orient → Decide → Act → Release as a horizontal stage indicator at the top of the card. Similar to a GitHub Actions progress bar but scoped to the OODA loop. The user always knows where in the synthesis pipeline they are.

Evidence rail as a visual element

The evidence rail should be visually distinct from the synthesis narrative — a left-bordered citation list, styled as the "grounding sources" that produced the output. Users should be able to expand individual citations. This is the anti-hallucination signal made visible.

Confidence and novelty as ambient metrics

Small, always-visible bars below each hypothesis or response. Not intrusive — ambient. They answer the silent question: "how much should I trust this?" without requiring the user to ask. This is the Wabi-Sabi "truth meter" concept made concrete.

Landing Page Design Principles

Product-as-hero, not illustration

Embed a live or animated synthesis card in the hero. Visitors should see a novelty score streaming, evidence rail populating, stages completing — the real system, not a mock-up of it. This is how Notion, Linear, and Framer all drive conversion now.

The Pearl Ladder as the differentiator

No competitor landing page talks about P(Y|do(X)). The Pearl Ladder visualization is MASA's strongest differentiating concept on a landing page. It answers "why isn't ChatGPT enough?" visually and memorably, without requiring the visitor to read an explanation.

Wabi-Sabi aesthetics = earned trust

The dark ink background, stone texture, bamboo accents, and generous whitespace signal something that SaaS blue-gradient pages don't: restraint and seriousness. The aesthetic itself communicates "this is not a toy." That's an implicit trust signal for a scientific tool.

One CTA, clearly above the fold

"Enter the Lab" as the clay-colored primary CTA. Secondary: "Read the white paper." Nothing else. Top SaaS pages in 2025 that converted best used the fewest CTAs. The current white paper page has no conversion action at all.

"Top SaaS landing pages now feature embedded product previews, video demos, and guided tours right in the hero section — users want to see how it works before signing up." — SaaSFrame, 2026

Ranked by impact-to-effort ratio, grounded in the Wabi-Sabi design system and research findings above.

    High impact Medium effort Design system: bamboo active state, stone bg
  • 2
    Main View: Stage progress bar for Hybrid mode Add the OODA loop stages (Observe → Orient → Decide → Act → Release) as a horizontal progress indicator at the top of every Hybrid session card. Users need to know where in the scientific pipeline they are. Low implementation cost, high epistemic value.

    High impact Low effort Wabi-Sabi: clay active stage, stone inactive
  • 3
    Main View: Ambient novelty + confidence meters Small inline progress bars below every hypothesis and response output. Not intrusive — ambient. This makes the trust signal structural, not textual. The Wabi-Sabi "Truth Meter" concept from the design system, applied to every output.

    High impact Low effort Causal blue / amber color coding by score
  • 4
    Landing Page: Pearl Ladder hero visualization Build the L1 / L2 / L3 stone pillar visualization as the landing page's centerpiece. Animate on scroll: L1 dims, L2 illuminates in bamboo gold. This is the clearest 5-second differentiator available to MASA that no competitor can copy.

    Very high impact Medium effort Scroll-triggered ink-bleed animation

    Very high impact Higher effort Can use static animation as Phase 1 shortcut
  • 6
    Main View: Evidence rail as a distinct visual component Style evidence citations as a left-bordered section beneath each response — visually distinct from the synthesis text. Each citation expandable. This is the anti-hallucination signal made structurally visible, not just mentioned in text.

    Medium impact Medium effort Clay left-border, monospace citation text

What not to do

Based on competitive analysis and the Wabi-Sabi design philosophy:

Don't add more CTAs to the sidebar. The sidebar is navigation, not conversion. Clutter kills the Ma (negative space) principle.

Don't use progress spinners for synthesis. Spinners communicate waiting. MASA should communicate work happening — stream the stages, not a spinner.

Don't center the chat input at the bottom like a generic chat tool. The input area in Causal Chat should be positioned as a query interface — ink stone aesthetic, query-first language, not "Send a message."

Don't make the landing page explain MASA — let it demonstrate MASA. Every paragraph of explanation is a conversion failure. The Pearl Ladder and the live synthesis card do the work words can't.
