Phase 6: Sovereign Memory ✓ Complete

Overview

The Sovereign Memory feature enables MASA to learn from past rejections via vector embeddings. When an idea is rejected by the auditor, its embedding is stored in a vector database. Before generating new ideas, the system queries for similar rejected patterns to avoid repeating mistakes.

Architecture

flowchart TD A["Novel Idea Generated"] --> B["generateEmbedding()"] B --> C{"checkRejection()"} C -->|"Similar Rejected Found"| D["Skip Idea"] C -->|"No Match"| E["MASA Audit"] E -->|"Approved"| F["saveIdeaEmbedding(isRejected=false)"] E -->|"Rejected"| G["saveIdeaEmbedding(isRejected=true)"] F --> H["idea_embeddings table"] G --> H H --> I["Vector Index (ivfflat)"] I --> C

Implementation Details

Component File Description
SQL Migration 20260120_vector_memory.sql Creates idea_embeddings table with pgvector, ivfflat index, and match_idea_embeddings RPC
Embedding Generator gemini.ts Uses Google's text-embedding-004 model (768 dimensions)
Vector Store Integration persistence-service.ts checkRejection() and saveIdeaEmbedding() methods
Pipeline Integration synthesis-engine.ts Lines 547-573: Filters ideas via priorRejectionCheckFn

Data Flow

  1. Generation: Novel ideas created by Claude
  2. Check: Each idea embedded and compared to rejected ideas (>90% similarity threshold)
  3. Filter: Similar ideas to rejected patterns are skipped
  4. Audit: Remaining ideas go through MASA Auditor
  5. Store: Embedding saved with is_rejected flag based on audit outcome
  6. Learn: Future generations avoid similar rejected patterns
Important: Apply the pgvector migration to your Supabase project to activate the feedback loop.

Phase 7: Chemical Entity Validation Validator ✓ Complete

Overview

Transitions MASA from Philosopher (logic only) → Scientist (hypothesis → simulation → evidence) by executing generated Python protocols in a secure Pyodide (WebAssembly) sandbox.

Architecture

flowchart TD A["Novel Idea"] --> B["ExperimentGenerator"] B --> C["protocolCode + labManual"] C --> D["ProtocolValidator"] D --> E{"Pyodide Sandbox"} E --> F["Execute Python"] F --> G["Capture stdout"] G --> H["Parse Metrics"] H --> I["ValidationResult"] I --> J["Attach to NovelIdea"] J --> K["Store in synthesis_results"]

Implementation Details

Component File Description
Pyodide Sandbox protocol-validator.ts WebAssembly sandbox with numpy, scipy, networkx
API Route validate-protocol/route.ts POST endpoint with security checks
Engine Integration synthesis-engine.ts Lines 669-692: validateProtocolFn callback
Type Definition types/index.ts validationResult field on NovelIdea
Stream Event streaming-event-emitter.ts protocol_validated event for real-time UI

Features

  • Secure Sandbox: Pyodide (WebAssembly) isolates Python execution - no filesystem access
  • Scientific Packages: numpy, scipy, networkx available for simulations
  • Metrics Parsing: Extracts p-values, Bayes factors from stdout
  • Timeout Protection: 30s max execution time prevents runaway simulations
  • Security Checks: Blocks dangerous imports (os, subprocess, socket, etc.)

Metrics Captured

Sample Size n: 1000 Simulation iteration count
Note: Pyodide loads ~15MB on first call. Subsequent validations reuse the cached instance for fast execution.
