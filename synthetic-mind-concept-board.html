01

The North Star Concept

Every current chat interface is a document metaphor — a list of messages stacked in a vertical feed, topped by a header, bottomed by an input. It's a text editor pretending to be a mind. We break this completely. The Synthetic Mind is organized around a central Shader Orb — a living, breathing Three.js shader animation that serves as the visual locus of intelligence. Five reasoning modes — Chat, Hybrid, Legal, Educational, Labs — orbit the orb in a slow, continuous ellipse. You don't click tabs. You intercept a mode as it passes.

The Metaphor Shift

From chat box to orbital brain. The interface communicates that this is an intelligence with presence, not a text relay. The orb breathes, pulses, and shifts color based on what it's doing.

The Spatial Grammar

Three vertical zones: Consciousness (the orb, 45%), Dialogue (thread, 35%), Command (input, 20%). Each zone has its own visual register and interaction logic.

The MASA Lineage

Inherits the warm cinematic dark lab soul — charcoal #0E0D0C, amber accent, Instrument Serif headings, JetBrains Mono labels. The shader orb adds spectral energy within that warm container.

05

Interaction Choreography

Mode Selection Flow
1

Orbital nodes auto-rotate

All five mode nodes circle the orb continuously at ~20s/revolution. The user observes the system in motion — no idle state.

autoRotate: true
2

User clicks a mode node

Rotation pauses. The clicked node transitions to its "12 o'clock" position via a smooth 700ms ease-spring arc. The other nodes coast to their relative positions.

autoRotate: false
3

Orb reacts

The shader's color tint transitions over 600ms. The active node expands 1.3×, gains a glowing border in its mode color. A hairline connector threads from the node to the orb center.

4

Dialogue zone updates

The AI-message left-border color transitions to the new mode accent. A tiny mode pill in the Command Zone updates. New messages inherit the new mode's context.

5

Return to orbit

Clicking the background or the active node again resumes auto-rotation. The orb slowly returns to its default blue-white state if no mode is locked.

autoRotate: true
Message Generation Flow
1

User submits message

User message materializes in the Dialogue Zone with a scale-up + fade-in entrance (origin: bottom center). Input field briefly dims.

2

Orb enters "generating" state

Shader velocity spikes. Pulse frequency doubles. The orb's glow radius expands subtly. A thin progress ring appears around the orb exterior.

velocity: 0.12
3

Response streams in

AI response card appears in Dialogue Zone with a cursor-style stream. The left accent border in mode color is present from first token. Cards have subtle `bg-3` frosted-glass surface.

4

Orb returns to idle

Velocity eases back to idle over 1.5s. Progress ring fades. The orb settles into its resting pulse.

velocity: 0.02
5

Evidence Rail updates

Confidence telemetry, causal citations, and provenance data update in the right rail — the existing MASA Evidence Rail pattern, unchanged.

Implementation Notes

Technical Considerations
Core
Shader Tinting via Uniform
Add a tintColor: vec3 uniform to the fragment shader. Mix the final gl_FragColor output with the tint using mix() at ~30% weight. Transition the tint using React state + lerp over ~600ms frames.
Core
Circular Clip Mask
The ShaderOrb renders inside a border-radius: 50% container with overflow: hidden. The Three.js canvas fills this container. No extra clip-path needed — WebGL respects CSS overflow on the canvas parent.
New
Velocity Control
Use a useRef for the target velocity and lerp the actual uniforms.time increment toward it each frame. This creates smooth acceleration/deceleration rather than jarring velocity jumps.
Adapted
Orbital Node Positioning
The existing calculateNodePosition() function from RadialOrbitalTimeline is used directly. The container needs absolute positioning relative to the ConsciousnessZone center, not the entire viewport.
New
Mode State via Context
A ModeContext React context provides activeMode, setMode, orbState, setOrbState to all three zones. This avoids prop-drilling between the ConsciousnessZone and DialogueThread.
