# Component Patterns

## Hero Module

- Keep one dominant statement and one quiet supporting line.
- Pair flat data with one textured visual block.
- Use asymmetric split layouts like 5/7 or 4/8 columns.

## Navigation

- Prefer calm top navigation with minimal items.
- Add progressive disclosure panels for advanced controls.
- Keep active state understated: underline or tonal block, no bright glow.

## Data Cards

- Present metric, trend hint, and causal note.
- Keep card chrome subtle; let content carry hierarchy.
- Use small tactile dividers instead of heavy borders.

## Forms

- Use high-clarity labels and one-line helper text.
- Keep field spacing generous to reduce cognitive load.
- Reserve strong accent color for validation and irreversible actions.

## Empty and Error States

- Keep language brief and actionable.
- Offer one next action and one secondary recovery path.
- Use grounded illustrations or geometric motifs, not mascots.

## Prompt Patterns for This Skill

- "Design a dashboard that feels like weathered intelligence, not glossy sci-fi."
- "Refactor this page into Wabi Sabi 2041 with calm typography and tactile depth."
- "Create a responsive component system that uses asymmetry without harming clarity."
